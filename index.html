This site is about web servers and how they shaped the internet as we know it. From the Cold War origins of ARPANET to the massive data-driven platforms of today, web servers are at the center of all of it.

Abstract

Communication has been something humans have chased for as long as there have been humans. It started with spoken language passed between people. Then came writing, which let information survive beyond a single conversation. The printing press, invented in 9th century China and later reinvented in Europe, let ideas reach thousands of people at once (History.com Editors, 2018). Each step made sharing knowledge faster and more accurate.

The World Wide Web is the latest and biggest step in that process. And sitting behind every website on the web is something called a web server. This site explores what web servers are, the two main types, and why they matter so much to the modern world.

Key Terms

Before getting into the content, here are some terms that come up a lot:

Term What It Means
Web Server A computer that stores files and sends them to users who request them over the internet.
ARPANET The government network built during the Cold War that eventually became the internet.
Static Web Server A server that always sends the same pre-built content to every visitor.
Dynamic Web Server A server that builds content on the fly based on each user's request.
HTML HyperText Markup Language. The code used to build the structure of web pages.
SQL Injection An attack method that inserts malicious code into a website's database queries.
Client The device or browser making a request to a server.
Database An organized system for storing and retrieving large amounts of data.

The History Behind It

The internet did not start as a public tool. It started as ARPANET, a defense project meant to keep military communications alive during a nuclear attack. Understanding that history explains a lot about how the web evolved into what it is today.

Read more on the concepts page

Two Types of Servers

Not all web servers work the same way. Static servers send fixed files and are fast and secure. Dynamic servers connect to databases and can serve personalized content. Both have their place, and knowing the difference matters.
